# Breakthrough Engine - Calibration Scorecard Template

## Scoring Scale (1-5)

| Score | Meaning |
|-------|---------|
| 1 | Very poor — no useful signal, would waste reviewer time |
| 2 | Poor — some structure but content is generic/obvious/wrong |
| 3 | Acceptable — meets minimum bar, some useful content |
| 4 | Good — genuinely interesting, would merit human review |
| 5 | Excellent — novel, well-supported, publication-worthy |

## Per-Run Scorecard

| Dimension | Score (1-5) | Notes |
|-----------|-------------|-------|
| **Novelty quality** | | Was the candidate genuinely novel? |
| **Plausibility** | | Is the hypothesis physically/scientifically plausible? |
| **Evidence quality** | | Did evidence items actually support the hypothesis? |
| **Simulation usefulness** | | Did simulation add meaningful signal? |
| **Publication worthiness** | | Would this deserve human attention as a publication? |
| **Operator review burden** | | How much effort to review? (5 = very low burden) |
| **Overall signal quality** | | Aggregate: was this run a good use of compute/time? |

## Additional Observations

- Was the result genuinely interesting? [yes/no]
- Were novelty heuristics too weak or too strict? [too weak / about right / too strict]
- Did retrieval add useful evidence or noise? [useful / noise / N/A]
- Would the draft deserve human attention? [yes / maybe / no]
